# Research notes

Create one Markdown file per meeting, experiment-review session, or major decision.

Suggested filename format:

```text
YYYY-MM-DD-topic.md
```

Each note should record:

1. Question or hypothesis.
2. Data and code version used.
3. Observations and figures.
4. Decision or next action.
5. Owner.

Do not put raw dataset files or credentials in this folder.
