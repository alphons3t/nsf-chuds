# Documentation and research notes

Use this folder for material that helps the team understand, reproduce, and explain the project:

- `project-brief.md` — challenge scope, data modalities, and evaluation criteria.
- `research-notes/` — dated meeting notes, observations, decisions, and source links.
- `methodology/` — model-design and validation notes as the approach develops.

Keep narrative notes in Markdown so every collaborator can review them in GitHub and reference them from the final report.
